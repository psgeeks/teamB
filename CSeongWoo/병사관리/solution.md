# 📝 병사 관리 풀이

## 1. 자료구조 설계

제한 시간 내에 대량의 병사 정보를 갱신, 조회하는 요구사항을 충족해야 하기 때문에, 정적 풀 배열로 병사를 관리하고, 이를 이중 연결리스트를 활용해 각각의 병사를 팀, 점수에 맞게 분류하여 연결하였습니다.

---

## 2. 설계와 그 의도

* **`static Node[] soldierArr = new Node[100001];`**
  
    * **의도:** 병사의 고유번호(`mID`)를 인덱스로 삼아, 해당 병사의 Node 객체 메모리 주소를 즉시 참조합니다.병사의 고유 번호는 1 ~ 100,000이고, 고용 횟수도 100,000회이며, 중복 고용은 존재하지 않기 때문에 배열은 단 한번만 초기화하고, 바로 해당 배열의 인덱스에 값을 저장합니다.
    * 시간복잡도: `O(1)`
* **`static Node[] nodes = new Node[100001];`**
    * **의도:** 새로운 병사를 영입할 때 사용할 객체 저장소(Pool)입니다. 최초 1회만 초기화 하고, `nodeCnt`를 init마다 0으로 초기화 시킵니다. 이를 통해 노드의 과도한 생성을 없앨 수 있습니다.
* **`static Node[][][] mTeams = new Node[5][5][2];`**
    * **의도:** `[팀(0~4)][점수(0~4)][0:head, 1:tail]` 구조를 가지는 3차원 배열입니다. 전체 병사를 25개의 그룹으로 나누어 관리함으로써, 특정 팀의 점수를 일괄 변경할 때 병사 개개인을 수정하지 않고 연결 리스트의 Head와 Tail 포인터만 교체하여 `O(1)`에 병합(Merge) 연산을 수행하기 위함입니다.

## 3. 각 함수별 동작 원리와 시공간 복잡도

모든 함수의 공간 복잡도는 전역으로 할당된 정적 배열을 재사용하므로, 추가적인 메모리 할당이 없어 실질적으로 **O(1)**에 수렴합니다. (전체 시스템의 공간 복잡도는 O(MAX_NODE)로 일정함)

### `UserSolution()` (생성자)
* **동작 원리:** 프로그램 최초 실행 시 객체 풀(`nodes`)에 10만 개의 Node를 미리 생성하고, `mTeams`의 각 그룹별 Head와 Tail 노드를 할당합니다.
* **시간 복잡도:** O(N) (N은 최대 병사 수, 최초 1회만 실행됨)

### `init()`
* **동작 원리:** 새로운 테스트 케이스가 시작될 때, 객체 풀의 인덱스를 가리키는 `nodeCnt`를 0으로 초기화하여 풀을 재사용할 수 있도록 세팅합니다.
* **시간 복잡도:** O(1)

### `hire(int mID, int mTeam, int mScore)`
* **동작 원리:** `nodes` 풀에서 새로운 노드를 가져와 초기화한 뒤, 0-based 인덱스로 변환한 팀과 점수에 해당하는 `mTeams` 연결 리스트의 Tail 바로 앞에 삽입합니다. 동시에 `soldierArr`에 등록하여 즉각적인 접근을 보장합니다.
* **시간 복잡도:** O(1)

### `fire(int mID)`
* **동작 원리:** `soldierArr`를 통해 해고할 병사의 노드에 직접 접근합니다. 해당 노드의 `prev`와 `next`를 서로 연결하여 현재 노드를 연결 리스트에서 분리하고, 배열에서도 참조를 지웁니다.
* **시간 복잡도:** O(1)

### `updateSoldier(int mID, int mScore)`
* **동작 원리:** `soldierArr`를 통해 대상 노드에 접근한 뒤, `fire` 함수와 동일한 원리로 현재 소속된 점수대 리스트에서 노드를 분리합니다. 이후 변경할 대상 점수대(`mScore`) 리스트의 Tail 포인터를 찾아 그 앞에 노드를 새롭게 편입시킵니다.
* **시간 복잡도:** O(1)

### `updateTeam(int mTeam, int mChangeScore)`
* **동작 원리:** 특정 팀의 모든 병사 점수를 일괄 갱신합니다. 병사 개개인을 순회하지 않고, 변경 전 점수대의 전체 리스트 덩어리를 변경 후 점수대의 Tail 뒤에 통째로 이어 붙이는(포인터 스와핑) 방식을 취합니다. 점수 증감 방향에 따라 순차 또는 역순으로 순회하여 덮어쓰기가 발생하지 않도록 제어합니다.
* **시간 복잡도:** O(1) (점수대 순회가 최대 5번으로 상수 시간에 고정됨)

### `bestSoldier(int mTeam)`
* **동작 원리:** 해당 팀의 최고 점수(4번 인덱스) 리스트부터 순차적으로 내려오면서 병사를 탐색합니다. 각 그룹 내에서 연결 리스트를 순회하며 가장 높은 `mID`를 찾아 반환합니다. 
* **시간 복잡도:** O(K) (K는 해당 팀의 최고 점수대에 속한 병사의 수. 최악의 경우 O(N)일 수 있으나, 그룹화 덕분에 평균적인 연산 횟수가 비약적으로 줄어듦)
