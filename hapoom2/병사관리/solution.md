# 사용한 자료구조

## 1. 팀별 / 점수별 연결리스트



```
head[team][score]
tail[team][score]
```

점수 범위가 1~5로 작기 때문에 각 팀마다 점수별 연결리스트를 따로 두고 병사를 저장합니다.

---

## 2. Lazy Deletion (version 사용)

- 연결리스트에서 특정 노드를 삭제하려면 보통 **해당 노드의 이전 노드를 찾아야 합니다.** 이를 위해 리스트를 순회해야 하므로 **O(N)의 탐색 비용** 이 발생할 수 있습니다.
- 병사 정보를 직접 삭제하거나 수정하지 않고, 병사의 상태가 바뀔 때마다 `version[id]`를 증가시켜 **최신 상태만 유효**하도록 관리합니다.

이렇게 하면 연결리스트 중간 삭제 비용을 제거할 수 있습니다.

---

# 함수별 정리

## 1. init()

### 코드 설명

- 병사 정보 배열을 초기화합니다.
- `version` 배열을 초기화합니다.
- 팀별 / 점수별 연결리스트의 `head`, `tail`을 생성합니다.
- 각 리스트는 더미 노드로 시작합니다.

---

## 2. hire(mID, mTeam, mScore)

- 새로운 병사를 고용하는 함수입니다.

### 코드 설명

- 병사의 현재 팀 정보를 저장합니다.
- `version[mID]`를 증가시킵니다.
- 해당 팀 / 점수 리스트의 뒤에 새 노드를 추가합니다.

### 설계 의도

- 병사의 상태를 직접 수정하는 대신, **최신 상태 노드를 새로 추가하는 방식**으로 관리합니다.
- 이 방식을 통해 연결리스트 중간 삭제 비용을 제거할 수 있습니다.
- 이후 상태 변경 시에도 같은 방식으로 처리할 수 있습니다.

---

## 3. add(id, team, score)

- 병사 노드를 해당 팀 / 점수 리스트의 맨 뒤에 추가합니다.

### 코드 설명

```
Nodenode=newNode(id,version[id]);
tail[team][score].next=node;
tail[team][score]=node;
```

- 새 노드를 생성합니다.
- 현재 tail 뒤에 연결합니다.
- tail을 새 노드로 갱신합니다.

### 설계 의도

- 리스트 삽입을 항상 O(1)에 처리하기 위함입니다.
- 이를 위해 `tail` 포인터를 함께 관리합니다.

---

## 4. fire(mID)

- 병사를 해고하는 함수입니다.

### 코드 설명

- 병사의 `version`을 무효값으로 바꿉니다.
- 연결리스트에서 직접 삭제하지는 않습니다.

### 설계 의도

- 연결리스트 중간 삭제를 피하고, **lazy deletion** 방식으로 처리하기 위함입니다.
- 삭제는 나중에 조회 시 최신 상태 여부로 걸러냅니다.

---

## 5. updateSoldier(mID, mScore)

- 병사 한 명의 점수를 변경합니다.

### 코드 설명

- `version[mID]`를 증가시킵니다.
- 새 점수에 해당하는 리스트에 새 노드를 추가합니다.
- 기존 노드는 그대로 둡니다.

### 설계 의도

- 기존 노드를 직접 찾아 수정하거나 삭제하지 않기 위함입니다.
- 즉, 점수 변경도 새 상태를 추가하는 방식**(lazy deletion)**으로 처리합니다.

---

## 6. updateTeam(mTeam, mChangeScore)

- 특정 팀 전체의 점수를 변경합니다.

### 코드 설명

- 각 점수 리스트가 변경 후 어느 점수로 가는지 계산합니다.
- 병사를 하나씩 옮기지 않고, 리스트 자체를 새 점수 칸에 이어붙입니다.

**핵심 코드**:

```
newTail[ns].next=head[mTeam][s].next;
newTail[ns]=tail[mTeam][s];
```

- 새 리스트 끝에 기존 리스트를 연결합니다.
- 새 리스트의 tail을 기존 리스트의 마지막 노드로 갱신합니다.

### 설계 의도

- 병사 수만큼 이동시키면 느리기 때문에, 리스트 단위로 재배치하도록 설계했습니다.
- 점수 범위가 1~5로 작아서 이런 방식이 가능합니다.

---

## 7. bestSoldier(mTeam)

- 해당 팀에서 가장 점수가 높은 병사를 찾습니다.
- 같은 점수라면 id가 가장 큰 병사를 반환합니다.

### 코드 설명

- 점수 5부터 1까지 내려가며 확인합니다.
- 각 리스트를 순회하면서 `version[id] == node.ver` 인 노드만 유효한 병사로 봅니다.
- 그중 가장 큰 id를 선택합니다.

### 설계 의도

- 리스트 안에는 이전 상태 노드가 남아 있을 수 있으므로, **최신 상태만 판별**합니다.
