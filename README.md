# 🧠 알고리즘 스터디

꾸준히 알고리즘 문제를 풀며 실력을 향상시키고 문제 해결 감각을 길러 유연하게 사고하는 것을 목표로 합니다.

<br />

## 🗓️ 진행방식

#### 0. B형 알고리즘은 조금 문제의 접근법을 설명하도록 구성하겠습니다.
- 다음의 폴더 구조를 따르며 PR을 작성하시면 됩니다!
```
📦 repository-root
 ┣ 📂 g0rnn
 ┃ ┣ 📂 병사관리
 ┃ ┃ ┣ 📄 solution.md
 ┃ ┃ ┗ 📄 Sol17141.java
 ┃ ┃ ┗ 📄 Sol17141_2.java
 ┃ ┃ ┗ 📄 mistake_log.md
 
```
  
- 반드시 4시간 동안 집중하여 문제를 풀기 바랍니다. B형을 준비할 때 며칠에 걸쳐 문제를 푸는 것은 안좋은 습관이라 생각합니다!
- 반드시 이클립스 디버거를 사용해보시길 바랍니다.
- 어떤 실수를 했는지 `mistake_log`를 만들어갔으면 합니다. 구현이 워낙 많기 때문에 어떤 부분에서 잦은 실수를 저지르는지 아는 것도 좋아보입니다. (필수는 아녜요)
```markdown
// mistake_log 예시

- 변수명 정하는데 10분 고민함
- 디버깅 어떻게 하는지 몰라서 구글 검색을 저질러버림;
- deep copy 안 해서 값 덮어씀
...

```
<br />

#### 1. 스터디는 2026-02-25부터 시작했으며, 매주 화요일까지 각자 1문제씩 제출
- 문제는 자율적으로 선택합니다. (프로그래머스, 백준, SWEA, 코드트리 등)

<br />

#### 2. 모든 스터디원은 주 1회 문제를 풀고 PR을 정성스레 작성하여 게시
- PR에는 문제 접근방식, 풀이방식, 사고의 흐름, 결론, 배운 점 등 자유롭게 내용을 구성하세요!
- 주차별로 브랜치를 만들어서 작업합니다. PR은 main을 향해 머지하세요~

<br />

#### 3. 다음 제출일이 되기 전까지 각자 다른 인원의 문제를 풀고 리뷰를 작성
- 화요일이 되면 모든 PR이 closed 되어야합니다!

<br />

#### 4. 리뷰 작성은 approved로 하고, 모든 인원이 리뷰를 완료했다면 merge!
- 리뷰를 approved로 하면 reviewer에 체크 표시가 떠서 확인하기 쉬워요.





<br />
<br />

### 2. 📝 커밋 메시지 규칙
