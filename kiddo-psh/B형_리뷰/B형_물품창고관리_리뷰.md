# B형 리뷰 : 물품 창고 관리

### 1. 제한사항

* Query <= 3,000 : 구간 내에서 가장 우선순위 높은 물품위치 최대 5개 반환
* get <= 10,000 : 특정 지점의 물품 수를 리턴
* set <= 10,000 : 특정 지점의 물품 수를 mValue로 바꿈
* work <= 40,000 : 구간 내 모든 물품의 양을 mDelta만큼 증가

* 전체 물품 창고 크기 N*N <= 500*500
* 물품 창고 구역 크기 N/K*N/K <= 20*20

### 2. 자료구조 설계

* i. `Item[][] items` : 좌표별 물품을 저장하는 배열로 크기 500*500으로 선언한다. 물품 객체에는 좌표와 물품량을 저장. set, get을 O(1)로 처리하게 해준다.
* ii. `TreeSet[][] sectors` : 구역별 물품리스트를 우선순위로 저장해둔다. 어차피 work는 구역별로 물품량이 변하기 때문에 treeset을 수정하지 않아도 우선순위가 변할 일은 없다. set의 경우는 우선순위가 바뀌기 때문에 treeset에서 제거 후 items로 물품객체에 접근해 물품량을 바꿔주고(delta값을 나중에 더해줄 것이기 때문에 감안하여 바꾼다.) treeset에 다시 넣는다.
* iii. `int[][] delta` : 구역별 물품량 변화량을 저장한다. 물품 객체에 접근하지 않고 물품 변화량만 누적하여 저장해두고 쓴다.   

### 3. 로직
* i. `work(Point A, Point B, mDelta)` 
	 *  A와 B의 좌표를 각각 K로 나눈다.
	 *  해당 구역을 돌며 delta 값에 mDelta를 더한다.

* ii. `get, set`
	*  해당 좌표의 객체에 접근해 물품량을 반환하고 물품량을 바꿀 수 있다.
	*  set 시 delta 값을 더해서 mValue가 될 수 있도록 `mValue-delta` 값으로 수정한다.
	
* iii. `query(Point A, Point B, mCount)`
	*  구역별 treeset에 앞부분 mCount개 만큼 꺼내서 delta값을 더한 후 리스트에 넣는다.
	*  list를 mCount 번 순회하며 우선순위 높은 객체를 찾는다.  


### 4. 나의 실수 : 메모리를 너무 아끼려고 했음
*   pq나 treeset 배열, 500*500배열을 활용했어야 했는데 격자 압축으로 풀 수 있을 줄 알고 20*20 배열만 고집했다. List<'Item'>[20][20] 만을 사용해서 delta 배열을 사용했음에도 query 시에는 모두 순회하며 delta배열만큼 갱신을 해줬어야 했음.
*  list 구현, 2차원배열 평탄화, 노드풀 사용 등 아는 방법 총 동원해봤지만 query를 완전 탐색한다는 것에서 시간초과를 면할 수 없었음.
