# B형 리뷰 : 국가행정

## 1. 제한사항
* 도시의 수 <= 10,000
* 각 도시 인구 수 <= 1,000
* `expand()` <= 5,000
* `calculate()` <= 2,500
* `divide()` <= 300

---

## 2. 자료구조 설계

* `int[]` -> population, roads, time : 인구 수 (0~N-1), 도로 수/이동시간 (0~N-2)
* `TreeSet<City>` 
  * 도로의 시간 내림차순, 도시 번호 오름차순 정렬 
  * 가장 긴 도로를 빠르게 찾기 위함.

---

## 3. 로직
* i. `expand()` 
  1. TreeSet에서 처음 원소를 뽑는다.
  2. 도로 수 증가, 이동시간 갱신한다.
  3. 갱신한 정보를 Set에 다시 추가.

* ii. `calculate()`
  * 경로는 단순 선형으로 구간 합을 구해준다.
  *  from ~ to 까지의 time 배열 값의 합 반환
  

* iii. `divide()` 
  * 이분탐색을 통해 최대값을 찾는다. -> 파라메트릭 서치 

* iv. `canDivide()`
  * limit 이하로 K개 이하 구간으로 나눌 수 있는가?
  * 그리디로 최대한 채우기 -> K초과하면 실패

---

## 4. 시간 복잡도

* i. `expand()`
  * `TreeSet` -> `O(log N)`
  * 최대 5,000회 -> **O(5000 log N)**
* ii. `calculate()`
  * 최악 `O(N)`
  * 최대 2,500회 -> **O(N*Q)**
* iii. `divide()`
  * 이분탐색 -> `O(log N)`
  * 내부 그리디 -> `O(N)`
  * 최대 300회 -> **O(300 * NlogN)**
