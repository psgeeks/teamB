## 아이디어

이 문제의 핵심은 **"어떻게 병사 입출력과 정보 갱신을 빠르게 하는가?"**

- 포인터 연산 : 병사 입출력 혹은 점수 갱신을 포인터를 활용해 O(1)의 속도로 수행
- 버킷 관리 : 병사들을 [팀][점수] 단위로 그룹화 시켜 관리

```java
	static Node[] soldiers = new Node[100001];
	static Node[][] head = new Node[6][6]; // mTeam * mScore
	static Node[][] tail = new Node[6][6]; // mTeam * mScore

	static class Node {
		int mID, mTeam;
		Node prev, next;
      ...
    }
```

**hire()**

- O(1)
- `soldiers` 배열에 해당 `mID`의 병사 객체가 없다면 새로 생성하고, 있다면 팀 정보만 갱신
- 동일한 인덱스의 요소의 `mID`는 변하지 않고, `mScore`은 연결 리스트로 관리하니 갱신할 필요 없음

**fire()**

- O(1)
- 연결 리스트에서 앞 뒤 노드의 포인터를 서로 연결 해줘서 자신을 리스트에서 빼낸다.
- 자신의 앞 뒤 노드 정보도 `null`로 초기화 한다.

**updateSoldiers()**

- O(1)
- 점수를 갱신할 병사의 팀을 미리 저장함.
- 저장된 팀에서 점수를 갱신할 병사를 `fire()` 한 후 갱신된 점수로 `hire()` 한다.

**updateTeam()**

- O(1) : `updateTeam()`안에서 포인터 4개만 바꾸면 됨
- 이하 병사의 점수를 `n`, 갱신할 점수를 `k`
- `k`가 0보다 클 경우
  - `n` 점수 리스트를 `n+k` 점수 리스트 뒤에 이어 붙인다.
  - `n = 4` 리스트부터 내림차순으로 진행해야 점수를 두 번 더하지 않음
  - `n = 5` 는 `n+k`가 항상 5이므로 갱신할 필요 없음
- `k`가 0보다 작을 경우
  - `n` 점수 리스트를 `n+k` 점수 리스트 뒤에 이어 붙인다.
  - `n = 1` 리스트부터 오름차순으로 진행해야 점수를 두 번 더하지 않음
  - `n = 1` 는 `n+k`가 항상 1이므로 갱신할 필요 없음

**bestSoldier()**

- O(N)
- 병사가 1명 이상 존재하는 가장 높은 점수 리스트에서 탐색함
