# 병사관리 실수 로그

## 자료구조 선택

처음에는 아래와 같이 `Map`을 사용했음

```java
	static Map<Integer, Soldier> soldiers;
	static Map<Integer, List<Soldier>> teams;
```

데이터 입출력이 오래걸려서 시간 초과가 났음

지금 생각해보면 데이터 구조 자체도 잘못 설계 되었음

## 자료 구조 설계

처음에는 두 개의 자료 구조를 사용해서 데이터를 관리했음

- 병사 전체를 저장하는 1차원 리스트
- 팀 단위로 병사를 저장하는 2차원 리스트

그린데 팀 점수를 업데이트 하거나 최고의 병사를 찾을 때 너무나 많은 반복을 돌아야 했음

그래서 팀 단위와 점수 단위를 모두 나눠서 3차원 리스트로 병사를 관리함
