## 아이디어

차량의 이동가능 거리가 베이스캠프 도착 후 회복된다는 점에서 차량의 이동가능 거리는 **두 정점을 간선으로 연결**할 수 있는지 판별하기 위한 조건이라 생각했음

차량은 간선으로 연결된 모든 정점을 순회하며 채굴된 광물을 모두 수거할 수 있음. 이 조건을 보고 **연결된 정점들끼리 하나의 서로소 집합**으로 만들고, 각 서로소 집합의 채굴량을 관리해야 한다고 생각했음

광물을 회수할 때 가장 우선순위가 높은 집합을 선택해서 차량을 떨어 뜨림. 집합의 우선순위는 집합 내 가장 우선순위가 높은 정점을 기준으로 정해진다. 집합 내 **가장 우선순위가 높은 정점을 루트로 정하면 집합의 루트끼리만 비교를 진행**하면 됨

## 자료구조

처음에는 정점 하나에 대한 모든 정보를 저장하는 클래스를 만들었는데, 클래스 배열을 관리할 때 속도가 너무 느려서 정점에 대한 모든 정보를 서로 다른 배열에 저장하도록 코드를 개선함

```java
int size; // 정점의 수

int r[] = new int[20000]; // row
int c[] = new int[20000]; // col
int q[] = new int[20000]; // quantity
int id[] = new int[20000]; // id

int parent[] = new int[20000];
int sum[] = new int[20000]; // 서로소 집합 내 채굴량의 합
```

## 구현

### init

`L` (차량 이동 거리/노드 연결 가능 거리)과 `size`(정점의 수)를 초기화 함

```java
void init(int L, int N) {
  this.L = L;
  this.N = N;
  size = 0;
}
```

### addBaseCamp

정점에 대한 정보를 배열에 저장하고, 저장된 모든 정점을 순회하면 연결가능한 정점과 `union`한다

### isBetter

`a` 정점이 `b`정점보다 우선 순위가 높다면 `true` 아니면 `false`를 반환함

### findSet

기존의 findSet과 동일한 로직인데, 호출 스택과 실행 시간을 줄이기 위해 재귀가 아닌 반복문으로 구현함

### union

우선순위가 높은 루트 노드에 낮은 루트노드를 자식으로 연결

### findBaseCampForDropping

자신의 부모가 자신은 노드(루트 노드)만 탐색함

루트 노드들 중 가장 우선순위가 높은 노드를 반환
