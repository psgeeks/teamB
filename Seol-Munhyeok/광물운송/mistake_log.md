## 실수 정리
* 버킷적 사고의 부족 -> 코드로 작성할 때 인덱스 실수가 많음 (더 많이 풀어봐야함)
* 우선순위 문제는 PQ나 TreeSet 말고 다른 방법도 있다.
  * 하나의 `better()` 함수를 만들고 union할 때, better인 원소를 `best` 배열에 저장하면 됨.
  *  루트 수 최대 20000, find 쿼리 최대 500 => 1000만이므로 선형 탐색으로 best를 찾아도 충분하다.
* Union-Find에서 parent, size, rank 말고 다른 요소를 추가할 수도 있음
  * 각 연결요소마다 들고있어야 하는 정보를 매 union 마다 추가하는 방식으로 문제를 풀 수 있음
 
* mId -> idx 매핑은 필요없고, idx -> mId 매핑만 필요하므로 HashMap 없이 배열로도 가능
* BaseCamp class 만들어도 되지만 배열로 전부 관리하는게 조금 더 효율적
